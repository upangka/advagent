# 🎛️ Effort 等级详解

这些选项对应不同的"思考强度"，用于平衡**质量**、**速度**和**成本**。

## 📊 Effort 等级对照表

| Effort 等级 | 核心机制与用途 | 适用场景 |
| :--- | :--- | :--- |
| **Low / Medium** | 快速响应模式：模型进行最精简的思考，以最快速度给出答案。 | 简单的代码补全、格式化、重命名变量等机械性任务。 |
| **High** | 默认均衡模式：在质量和速度上取得最佳平衡。<br>⚠️ 注：有开发者反馈，Anthropic 曾将默认值从 High 调整为 Medium，可能需要手动改回 High。 | 日常开发任务：编写新功能、编写单元测试、常规重构。 |
| **xHigh** | 深度推理模式：进行更长的"思维链"推理，解决更复杂的问题。 | 调试棘手的并发问题、跨多文件的架构设计、复杂的代码审查。 |
| **Max** | 最大算力模式：调用几乎全部推理预算，力求在最具挑战性的问题上"一击必中"。 | 安全审计、设计核心算法、制定复杂的迁移计划。 |
| **Ultracode** | 自动编排模式：触发 Dynamic Workflows 功能，生成 JavaScript 脚本并调度最多 1000 个子智能体并行工作。 | 超大型任务：全代码库框架迁移、全面安全漏洞扫描、多源调研。 |

## 💡 简单总结

| 类别 | 包含等级 | 核心逻辑 |
| :--- | :--- | :--- |
| **渐变推理刻度** | Low → Medium → High | 从快到慢、从浅到深 |
| **超频模式** | xHigh → Max | High 之上的加强版，解决常规思考搞不定的难题 |
| **外挂模式** | Ultracode | 改变工作方式：从单兵作战 → 大军团并行 |

## ⚙️ 效率与成本平衡指南

### 设置方法
在 Claude Code 中使用 `/effort` 命令即可切换等级。

### 推荐实践

1.  **日常默认使用 `High`**
    - 绝大多数编程任务的最佳成本效益比

2.  **遇到难题切换到 `xHigh`**
    - 当 Claude 几次都无法解决某个 bug
    - 当感觉给出的方案太浅显时
    - 执行 `/effort xhigh` 让它深度思考

3.  **海量任务使用 `Ultracode`**
    - 任务范围是整个项目时
    - 在提示词前加上 `ultracode` 关键词
    - Claude 会自动创建动态工作流
